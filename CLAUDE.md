## Architecture

This is a **Wails v2** desktop application: Go backend + embedded WebKit frontend.

### Backend (Go)

- **`main.go`** ‚Äî Wails entry point; embeds frontend assets, configures window
- **`app.go`** ‚Äî Core `App` struct exposed to frontend via Wails bindings; handles connection CRUD, starts the WebSocket server, and exposes `GetWSPort()` so the frontend can connect to it
- **`internal/config/`** ‚Äî Reads/writes `~/.multi-ssh/connections.yaml` (YAML); provides CRUD for connections
- **`internal/model/`** ‚Äî `Connection` and `Settings` structs; `Connection.SSHArgs()` builds the argument list for the system `ssh` command
- **`internal/ssh/`** ‚Äî Validates SSH connectivity (`TestConnection`)
- **`internal/pty/`** ‚Äî PTY session manager; spawns `ssh`/`sshpass` with a PTY, bridges I/O over WebSocket; handles terminal resize via binary frame prefix `\x01<cols>,<rows>`; platform-specific code for Unix vs Windows

### Frontend (Vanilla JS/HTML/CSS)

- **`frontend/app.js`** ‚Äî All UI logic: state (`connections[]`, `selectedIndex`, `currentSessionId`, `term`), sidebar list with group/search, connection form modal, and xterm.js terminal lifecycle
- **`frontend/style.css`** ‚Äî Apple-inspired dark theme; CSS variables for colors/spacing
- **`frontend/vendor/`** ‚Äî Vendored xterm.js (v5) with fit and web-links addons
- **`frontend/wailsjs/`** ‚Äî Auto-generated JS bindings; **do not edit manually** ‚Äî regenerated by `wails dev/build`

### Data Flow

1. Frontend calls Go methods via auto-generated Wails bindings (e.g., `window.go.main.App.StartSSHSession(id)`)
2. Go starts a PTY subprocess (`ssh ‚Ä¶`) and a local WebSocket server on a random port
3. Frontend connects xterm.js to `ws://localhost:<port>/ws/terminal?id=<sessionId>`
4. PTY I/O is bridged bidirectionally over the WebSocket

### Configuration

Stored at `~/.multi-ssh/connections.yaml`. Auto-created on first run with 0700 directory permissions. Port `0` on a connection means "use the default (22)".
